# github-flow-helper

GitHub Flowの標準的な開発ライフサイクル（メインブランチの保護、トピックブランチでの作業、検証、プルリクエスト（PR）の作成、マージ後のクリーンアップ）を安全かつ円滑に進めるためのスキルです。

## 1. Git運用ルールとブランチ戦略

GitHub Flowの原則に従い、本番コード（`main`）の安定性を保ちながら機能開発を進めます。

### 禁止事項
* **メインブランチの保護**:
  - `main`ブランチへの直接の `git commit` および `git push` は絶対に行わないでください。
  - ローカルでの `main` ブランチへの直接のマージ（`git merge`）も禁止です。
* **マージの承認**:
  - `main` ブランチへのマージは、必ず GitHub 上のプルリクエスト（PR）を介し、ユーザー自身による承認・マージ実行のもとで行われなければなりません。AIエージェント自身が直接 `main` へのマージを完了させてはなりません。

### 許可事項 (自動実行可能)
* **トピックブランチへの操作**:
  - `main` ブランチ以外の開発用・トピック用ブランチ（例: `feat/xxx`, `fix/xxx`）における `git commit` および `git push` は、ユーザーの個別の許可を待まわずに自動実行して構いません。
* **読み取り専用コマンド**:
  - リポジトリの状態を確認するため、以下の読み取り専用コマンドは自動実行できます: `git status`, `git log`, `git diff`, `git branch`

### コミットメッセージのルール
変更内容を簡潔に表すメッセージを指定し、適切なプレフィックスを付与してください。
* `feat:` 新機能の実装や追加
* `fix:` バグ修正や挙動の調整
* `clean:` リファクタリングや不要になったコード/機能の削除
* `docs:` ドキュメントやルールファイルの作成・更新
* `test:` ユニットテストの追加・変更

## 2. 検証とプルリクエスト (PR) 作成

### 検証の義務
* PRの作成（およびリモートへのプッシュ）を行う前に、必ず検証用スクリプト `skills/github-flow-helper/scripts/verify.ps1` （またはプロジェクトに定義されたGradleタスク/npmスクリプト等）を実行し、すべて正常にパスすることを確認してください。

### PR作成手順
1. **作業ブランチの作成**: 現在 `main` ブランチにいる場合は、`main` から分岐した新しい作業ブランチ（例: `feat/xxx`, `fix/xxx`）を作成して移動します。
2. **検証の実行**: `skills/github-flow-helper/scripts/verify.ps1` などの検証コマンドを実行し、エラーがないことを確認します。
3. **コミットとプッシュ**: 適切なプレフィックスを付けたメッセージでコミットし、リモートにプッシュします。
4. **プルリクエストの作成**: `gh pr create` コマンドを使用してPRを作成します。タイトル・本文は日本語。
5. **本文 (Body) 構成**:
   - 変更の目的（背景）
   - 主な変更内容（ファイルの箇条書きなど）
   - 関連する Issue があればそのリンク（例: `Closes #Issue番号`）
6. **完了報告**: 作成されたPRのURLをユーザーに報告します。

## 3. マージ後のクリーンアップ

ユーザーによるPRのマージが完了したことを確認したら、以下の操作を行ってローカル環境を最新化し、不要になったブランチを削除して整理します。
1. **メインブランチへ移動**: `git checkout main`
2. **最新の取得**: `git pull origin main`
3. **ローカルブランチの削除**: `git branch -d <topic-branch>`
4. **リモート追跡ブランチのクリーンアップ**: `git fetch --prune`

## 4. 開発とフィードバック

このスキルは [asabon/ag-developer-skills](https://github.com/asabon/ag-developer-skills) で開発されています。
バグ報告や機能の改善提案（スクリプトの追加など）がある場合は、以下のルールに従って開発元リポジトリの [Issues](https://github.com/asabon/ag-developer-skills/issues) に起票してください。

### 起票時の必須ルール（AIが自動適用できるようにするため）
* **対象ファイルの明記**: 修正・追加対象となるファイルの相対パスを明記してください。
* **具体的な修正案**: 抽象的なテキストだけでなく、修正箇所の「変更前・変更後のコードスニペット」または「追加するスクリプトコード」をマークダウンのコードブロックで記述してください。
* **検証方法の提示**: 修正が正しく適用されたかを判定するための確認手順（チェックリスト）を記述してください。
