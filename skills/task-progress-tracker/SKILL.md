# task-progress-tracker

エージェントの作業状況を適切にドキュメント化して保存・復旧し、プロジェクトの進捗を `task.md` によって一貫した形式で管理・標準化するためのスキルです。

## 1. タスク管理ルール (`task.md`)

`docs/progress/task.md` ファイルは、プロジェクト全体の進捗状況を表す「唯一の真実のソース (Source of Truth)」です。

### 運用ルール
* **進捗の同期**:
  - `docs/progress/task.md` ファイルは、常にプロジェクトの最新状態を反映している必要があります。
  - **最新の状況を把握するために、作業開始前に必ず参照してください。**
  - 以下のタイミングでは、必ず `docs/progress/task.md` を更新してください：
    - 新しいタスクやアクションアイテムを特定したとき。
    - 既存のタスクが完了したとき。
    - タスクの優先順位やステータスが変更されたとき。
* **タスクの記述形式**:
  - `[ ]` 未完了のタスク
  - `[/]` 進行中のタスク
  - `[x]` 完了したタスク

### ファイル構成テンプレート
`docs/progress/task.md` は以下の構成で記述してください。
```markdown
# Task: [タスク名/Issue名]

## 概要 (Goal Description)
タスクの目的、背景、およびゴールについて簡潔に説明します。

- [ ] セクション名 <!-- id: 0 -->
    - [ ] タスク項目 <!-- id: 1 -->
```

---

## 2. 保存ワークフロー (`/save`)

`/save` コマンドが実行された際、または現在のセッションを終了して進捗を保存する際は、以下のワークフローを実行します。

### 手順
1. **履歴の確認**: 最近の対話履歴と実行されたツールを確認します。
2. **進捗ファイルの更新**: 現在のステータスを正確に反映するため、`docs/progress/task.md` を更新します。
   - 完了したタスクに `[x]` をマークします。
   - 進行中のタスクに `[/]` をマークします。
   - セッション中に特定された新しいタスクやアクションアイテムを追加します。
3. **報告**: 保存した進捗状況をユーザーに簡潔に報告します。

---

## 3. 再開ワークフロー (`/resume`)

`/resume` コマンドが実行された際、または中断された作業を再開する際は、以下のワークフローを実行します。

### 手順
1. **進捗の読み込み**: タスク進捗ファイルを読み、現在のステータスを正しく理解します。
2. **ファイルの表示**: `view_file docs/progress/task.md` を実行して、直接内容を確認します。
3. **現状の報告**: 現在のステータスと未完了のタスクをまとめ、日本語でユーザーに報告します。
